1. 体験会機能の全体像（MVCの流れ）
MVC とは？
Laravel は「Model（モデル）」「View（ビュー）」「Controller（コントローラー）」の3つに役割を分けて作る仕組みです。
① ブラウザから URL にアクセスする → ② ルートがコントローラーに振り分ける → ③ コントローラーがモデルでデータを取得 → ④ ビューでHTMLを組み立てて返す

2. データベーステーブル
テーブルとは？
データを保存する Excelのような表のことです。行が1件のデータ、列がデータの項目（カラム）です。

■ trial_events（体験会テーブル）

カラム名型説明
idbigIncrements自動で採番される体験会のID（主キー）
event_datedate開催日
start_timestring開始時間（例: 10:00）
end_timestring終了時間（例: 12:00）
course_namestringコース名（A / B / C）
capacityinteger定員数
reserved_countinteger現在の予約人数
statusinteger0 = 空きあり、1 = 満員
created_bybigInteger登録した担当者のユーザーID
created_at / updated_attimestamp作成日時・更新日時（Laravel が自動で入れる）

■ trial_reservations（体験会予約テーブル）

カラム名型説明
idbigIncrements予約ID（主キー）
inquiry_idbigInteger（外部キー）どの問い合わせからの予約か
trial_event_idbigInteger（外部キー）どの体験会への予約か
reservation_statusstring予約状態（例: 予約済み）
reserved_atdatetime予約した日時
checked_in_atdatetime（nullable）受付した日時（来たら記録）
attendance_statusstring（nullable）出席状態
notetext（nullable）備考
created_bybigInteger（nullable）登録した担当者ID
外部キー（Foreign Key）とは？
別のテーブルのIDを参照する仕組みです。inquiry_id は inquiries テーブルの id を指し、trial_event_id は trial_events テーブルの id を指します。これにより「この予約は○○の問い合わせのもの」「この体験会への予約」という関係が表現できます。
3. モデル（Model）
モデルとは？
データベースのテーブル1つに対して1つのモデルクラスを作ります。
モデルを使うと、SQLを直接書かなくても TrialEvent::all() のようにPHPのコードでデータを取得・保存できます。
app/TrialEvent.php 全ロール共通

体験会（trial_events テーブル）を操作するモデルです。

class TrialEvent extends Model
{
    // ① 保存を許可するカラムの一覧（セキュリティのための設定）
    protected $fillable = [
        'event_date', 'start_time', 'end_time',
        'course_name', 'capacity', 'reserved_count',
        'status', 'created_by',
    ];

    // ② 状態の表示ラベル（0 → '空きあり'、1 → '満員'）
    const STATUS_LABELS = [
        0 => '空きあり',
        1 => '満員',
    ];

    // ③ event_date を日付オブジェクトとして扱う → format() が使えるようになる
    protected $dates = ['event_date'];

    // ④ この体験会に紐づく予約一覧を取得するリレーション（1対多）
    public function reservations()
    {
        return $this->hasMany(TrialReservation::class);
    }
}
① $fillable ― ここに書いたカラムだけ create() や fill() で値をセットできます。書いていないカラムには外部から値を入れられないので、セキュリティ対策になります。
③ $dates ― 指定したカラムを Carbon（日付専用クラス）として扱います。これで $event->event_date->format('Y/n/j') のように日付フォーマットが使えます。
④ hasMany（1対多） ― 体験会1件に対して予約が複数ある関係を表します。$trialEvent->reservations で紐づく予約一覧を取得できます。

4. コントローラー ― TrialEventControllerapp/Http/Controllers/TrialEventController.php
コントローラーとは？
ページごとの処理（データ取得・保存・バリデーション）をまとめた司令塔です。1つのページ = 1つのメソッドが基本です。
アクセス制限の仕組み
// ① すべてのメソッドでログインを必須にする
public function __construct()
{
    $this->middleware('auth');
}

// ② 講師はアクセス不可（受付・管理者だけOK）
private function checkRole()
{
    if (auth()->user()->role === 'instructor') {
        abort(403);
    }
}

// ③ 管理者だけアクセス可能
private function checkAdmin()
{
    if (auth()->user()->role !== 'admin') {
        abort(403);
    }
}
① middleware('auth') ― ログインしていない人はこのコントローラーのどのページも開けません。
②③ 各メソッドの先頭で $this->checkRole() または $this->checkAdmin() を呼ぶことで、ロールに応じたアクセス制限をかけます。

store() ― 新規登録処理のポイント
public function store(Request $request)
{
    $this->checkRole(); // ① 講師はここで止まる

    // ② バリデーション（入力チェック）
    $request->validate([
        'event_date'  => 'required|date',    // 必須・日付形式
        'start_time'  => 'required',         // 必須
        'course_name' => 'required|string|max:100',
        'capacity'    => 'required|integer|min:0', // 0以上の整数
    ]);

    // ③ DBに保存する
    TrialEvent::create([
        'event_date'  => $request->event_date,
        'created_by'  => auth()->id(), // ログイン中のユーザーID
    ]);

    // ④ 一覧ページにリダイレクト（フラッシュメッセージ付き）
    return redirect()->route('trial.index')
        ->with('success', '体験会を登録しました。');
}
② validate() ― フォームの入力値をチェックします。エラーがあれば自動で前のページに戻り、エラーメッセージが表示されます。
③ TrialEvent::create() ― $fillable に書いたカラムに一括で値をセットしてDBに保存します。
④ redirect()->route()->with() ― 指定したルートにリダイレクトし、セッションにフラッシュメッセージを乗せます。ビュー側で session('success') で受け取れます。

// ② 確認ページ表示（DBには保存しない）
public function confirm(Request $request, Inquiry $inquiry)
{
    $this->checkRole();
    $request->validate([
        'trial_event_id' => 'required|exists:trial_events,id',
        // exists: DBに実際に存在するIDかチェックする
    ]);

    $trialEvent = TrialEvent::findOrFail($request->trial_event_id);
    return view('trial.experience_reservation_conf',
        compact('inquiry', 'trialEvent'));
}

// ③ 予約をDBに保存する
public function store(Request $request, Inquiry $inquiry)
{
    TrialReservation::create([
        'inquiry_id'         => $inquiry->id,
        'trial_event_id'     => $request->trial_event_id,
        'reservation_status' => '予約済み',
        'reserved_at'        => now(), // 現在日時
        'created_by'         => auth()->id(),
    ]);
    return redirect()->route('trial.index')
        ->with('success', '体験会を予約しました。');
}

// 予約取り消し（管理者のみ）
public function destroy(TrialReservation $reservation)
{
    $this->checkAdmin();
    $trialEventId = $reservation->trial_event_id; // 戻り先のIDを先に控えておく
    $reservation->delete();
    return redirect()->route('trial.show', $trialEventId)
        ->with('success', '予約を取り消しました。');
}
6. ルート（Routes） ― web.php
ルートとは？
「このURLにアクセスしたらこのコントローラーのこのメソッドを呼ぶ」という対応表です。
Route::get('/url', 'Controller@method')->name('名前') という形で書きます。
routes/web.php ― 体験会関連ルート
// 体験会予約（inquiry に紐づく）
Route::get ('/inquiry/{inquiry}/trial-reservation',         '...@create' )->name('trial.reservation.create');
Route::post('/inquiry/{inquiry}/trial-reservation/confirm',  '...@confirm')->name('trial.reservation.confirm');
Route::post('/inquiry/{inquiry}/trial-reservation',          '...@store'  )->name('trial.reservation.store');
Route::delete('/trial-reservations/{reservation}',           '...@destroy')->name('trial.reservation.destroy');

// 体験会（静的ルートを {trialEvent} より先に書く ← 重要！）
Route::get   ('/trial-events',                     '...@index'        )->name('trial.index');
Route::get   ('/trial-events/create',              '...@create'       )->name('trial.create');
Route::get   ('/trial-events/calendar',            '...@calendar'     )->name('trial.calendar');
Route::post  ('/trial-events',                     '...@store'        )->name('trial.store');
Route::get   ('/trial-events/{trialEvent}',         '...@show'         )->name('trial.show');
Route::get   ('/trial-events/{trialEvent}/edit',    '...@edit'         )->name('trial.edit');
Route::post  ('/trial-events/{trialEvent}/confirm', '...@confirmEdit'  )->name('trial.confirmEdit');
Route::put   ('/trial-events/{trialEvent}',         '...@update'       )->name('trial.update');
Route::get   ('/trial-events/{trialEvent}/delete',  '...@confirmDelete')->name('trial.confirmDelete');
Route::delete('/trial-events/{trialEvent}',         '...@destroy'      )->name('trial.destroy');
注意 静的ルートは {パラメータ} より上に書く ― /trial-events/calendar を /trial-events/{trialEvent} より後に書くと、"calendar" が trialEvent のIDとして解釈されてしまいます。
注意 PUT / DELETE はHTMLフォームで直接送れない ― Blade で @method('PUT') / @method('DELETE') を書くと、Laravel が内部的に変換してくれます。

Blade テンプレートとは？
